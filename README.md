## Overview

This handbook is a **comprehensive cybersecurity foundation guide** designed in a rigorous, concept-first style similar to top-tier technical training environments. It focuses on **defensive cybersecurity**, helping learners understand how digital systems work, how threats emerge, and how individuals and organizations can protect their assets.

It is suitable for:
- training institutes
- self-paced learners
- beginner cybersecurity cohorts
- academic support programmes
- technical bootcamps

---

## Programme Goals

By the end of this handbook, learners should be able to:

- understand the core purpose of cybersecurity
- explain the **CIA triad** clearly
- identify digital assets, vulnerabilities, threats, and risks
- describe how networks function and why they matter in security
- work confidently with basic Windows and Linux security concepts
- understand common cyber threats and attack terminology
- apply strong digital hygiene practices in real life
- complete beginner-level labs and mini projects

## CYB101 — Introduction to Cybersecurity

| Item | Details |
|---|---|
| **Course Code** | CYB101 |
| **Course Title** | Introduction to Cybersecurity |
| **Focus** | Overview of cybersecurity concepts, domains, and attack surfaces |

### Course Purpose

This module introduces learners to the field of cybersecurity and explains why protecting systems, networks, applications, and data is essential in a digital world.

### Learning Outcomes

Learners should be able to:
- define cybersecurity
- explain confidentiality, integrity, and availability
- identify major cybersecurity domains
- explain attack surfaces
- distinguish threat, vulnerability, and risk

### Core Concepts

#### 1. What is Cybersecurity?
Cybersecurity is the protection of systems, devices, networks, and data from unauthorized access, disruption, alteration, or destruction.

#### 2. The CIA Triad
- **Confidentiality** — protecting data from unauthorized access
- **Integrity** — ensuring data remains accurate and untampered
- **Availability** — ensuring systems and data are accessible when needed

#### 3. Threats, Vulnerabilities, and Risk
- **Threat** — something capable of causing harm
- **Vulnerability** — a weakness that can be exploited
- **Risk** — the probability and impact of exploitation

#### 4. Major Cybersecurity Domains
- network security
- endpoint security
- cloud security
- application security
- governance, risk, and compliance
- identity and access management
- incident response
- security awareness

#### 5. Attack Surfaces
Examples include:
- laptops
- mobile phones
- email accounts
- Wi-Fi networks
- websites
- cloud dashboards
- removable storage devices
- human users

### Practical Activities
- List 10 digital assets in a school, office, or home setup.
- Identify one threat and one security control for each asset.
- Build a simple risk register with: Asset, Threat, Vulnerability, Impact, Control.

### Assessment Questions
1. Explain the CIA triad with real-life examples.
2. What is the difference between threat and vulnerability?
3. Why is cybersecurity not only a technical issue?
4. Mention five cybersecurity domains.
5. Give five examples of attack surfaces.

## CYB102 — Networking Fundamentals

| Item | Details |
|---|---|
| **Course Code** | CYB102 |
| **Course Title** | Networking Fundamentals |
| **Focus** | Network architecture, TCP/IP, DNS, firewalls, and routing |

### Course Purpose

This module explains how devices communicate across networks and why networking knowledge is essential for cybersecurity.

### Learning Outcomes

Learners should be able to:
- explain what a network is
- describe LAN, WAN, internet, and intranet
- explain TCP/IP basics
- describe DNS, DHCP, routers, switches, and firewalls
- understand ports, protocols, and routing

### Core Concepts

#### 1. What is a Network?
A network is a group of connected devices that communicate and share resources.

#### 2. Types of Networks
- **LAN** — Local Area Network
- **WAN** — Wide Area Network
- **Internet** — Global public network
- **Intranet** — Internal private network

#### 3. TCP/IP Basics
- **TCP** provides reliable delivery
- **IP** handles addressing and routing
- data moves in packets across networks

#### 4. DNS
DNS translates domain names into IP addresses so devices can locate services online.

#### 5. Network Devices
- **Switch** — connects devices within a local network
- **Router** — connects different networks
- **Firewall** — filters traffic based on rules

#### 6. Ports and Protocols
Examples include:
- HTTP
- HTTPS
- DNS
- SSH
- FTP
- SMTP

### Practical Activities
- Draw a home or office network diagram.
- Use `ipconfig` or `ip addr` to identify system network details.
- Use `ping` to test connectivity.
- Use `tracert` or `traceroute` to observe the path to a website.
- Use `nslookup` to inspect DNS resolution.

### Assessment Questions
1. What is the difference between TCP and IP?
2. What does DNS do?
3. What is the role of a router?
4. Why are firewalls important?
5. Explain the journey of a packet from a laptop to a website.

## CYB103 — Operating Systems & Security Basics

| Item | Details |
|---|---|
| **Course Code** | CYB103 |
| **Course Title** | Operating Systems & Security Basics |
| **Focus** | Windows and Linux environments, permissions, and file systems |

### Course Purpose

This module introduces learners to the role of operating systems in cybersecurity, with a strong emphasis on user privileges, file permissions, patching, and system hardening.

### Learning Outcomes

Learners should be able to:
- explain what an operating system does
- compare Windows and Linux basics
- understand users, groups, and permissions
- explain files, folders, and file systems
- describe the importance of patching and updates
- explain least privilege and basic hardening

### Core Concepts

#### 1. What an Operating System Does
An operating system manages:
- hardware resources
- memory
- processes
- storage
- user accounts
- permissions
- application execution

#### 2. Windows vs Linux
- **Windows** is common on office desktops and enterprise endpoints.
- **Linux** is widely used for servers, cloud systems, and technical security labs.

#### 3. Users, Groups, and Privileges
Security depends heavily on controlling who can access what. Permissions help enforce boundaries.

#### 4. Least Privilege
Users should only have the minimum access required to perform their duties.

#### 5. File Systems and Permissions
Students should understand:
- paths
- ownership
- read, write, execute permissions
- system files vs user files

#### 6. Patching and Hardening
Basic hardening includes:
- applying updates
- disabling unused services
- reducing admin access
- locking systems when idle
- using endpoint protection
- reviewing logs

### Practical Activities
- Create and compare standard user and admin accounts.
- Practice Linux terminal basics: `pwd`, `ls`, `cd`, `mkdir`, `cp`, `mv`, `cat`.
- Identify startup applications and unnecessary services.
- Check installed updates on a Windows or Linux system.
- Explain the risk of over-permissive file access.

### Assessment Questions
1. What is the difference between authentication and authorization?
2. Why is least privilege important?
3. Compare Windows and Linux in one paragraph.
4. What are file permissions?
5. Why is patching a major security practice?

## CYB104 — Cybersecurity Terminologies & Threat Landscape

| Item | Details |
|---|---|
| **Course Code** | CYB104 |
| **Course Title** | Cybersecurity Terminologies & Threat Landscape |
| **Focus** | Malware, phishing, ransomware, insider threats, APTs, botnets, attack vectors |

### Course Purpose

This module gives learners a strong vocabulary and working understanding of the modern cyber threat environment.

### Learning Outcomes

Learners should be able to:
- define common cybersecurity terms accurately
- classify common cyber threats
- identify phishing and social engineering techniques
- explain malware categories
- describe insider threats, APTs, and botnets
- relate threats to attack vectors and defensive controls

### Core Concepts

#### 1. Malware
Malware refers to malicious software designed to damage, disrupt, spy on, or control systems.

Examples:
- virus
- worm
- trojan
- spyware
- ransomware
- keylogger

#### 2. Phishing
Phishing is a deceptive attempt to trick users into revealing information or performing risky actions.

Types include:
- email phishing
- spear phishing
- smishing
- vishing

#### 3. Insider Threats
An insider threat comes from someone with legitimate access who causes harm intentionally or accidentally.

#### 4. APTs
Advanced Persistent Threats are usually sophisticated, targeted, and long-term intrusions.

#### 5. Botnets
A botnet is a group of compromised devices controlled remotely by an attacker.

#### 6. Attack Vectors
Examples include:
- malicious email
- weak credentials
- exposed remote access
- vulnerable software
- infected USB drives
- fake websites

### Practical Activities
- Review a safe mock phishing email and identify warning signs.
- Classify examples into malware, phishing, insider threat, or network attack.
- Create a cybersecurity glossary notebook with term, meaning, example, impact, and defense.
- Build an attack-chain diagram showing how one weakness can lead to compromise.

### Assessment Questions
1. Define ransomware.
2. What is the difference between phishing and spear phishing?
3. What is an insider threat?
4. What makes an APT different from a random attack?
5. Mention five attack vectors.

## CYB105 — Security Mindset & Digital Hygiene

| Item | Details |
|---|---|
| **Course Code** | CYB105 |
| **Course Title** | Security Mindset & Digital Hygiene |
| **Focus** | Security awareness, password hygiene, MFA, secure browsing, phishing identification |

### Course Purpose

This module teaches the human side of cybersecurity and focuses on practical behaviors that reduce risk in daily digital life.

### Learning Outcomes

Learners should be able to:
- explain cyber hygiene clearly
- apply stronger password practices
- explain the importance of MFA
- identify suspicious links and phishing indicators
- practice safer browsing behavior
- create a personal digital safety checklist

### Core Concepts

#### 1. Security Mindset
A security mindset means pausing to verify before trusting. Learners should ask:
- Who sent this?
- Why am I receiving this request?
- Is the link legitimate?
- What is the risk if I proceed?

#### 2. Password Hygiene
Strong password behavior includes:
- long passwords
- unique passwords for each service
- no password reuse
- using a password manager where possible

#### 3. Multi-Factor Authentication (MFA)
MFA adds extra protection beyond a password, reducing the damage from stolen credentials.

#### 4. Secure Browsing
Safer browsing includes:
- checking URLs carefully
- avoiding suspicious downloads
- using trusted platforms
- updating browsers regularly
- watching for fake login pages

#### 5. Phishing Identification
Warning signs include:
- urgency
- unexpected attachments
- suspicious links
- requests for passwords or money
- poor spelling or branding inconsistencies

#### 6. Digital Hygiene Practices
- keep software updated
- back up important files
- lock devices
- avoid unsafe public systems for sensitive work
- review account activity regularly
- report suspicious behavior quickly

### Practical Activities
- Create a strong password workshop.
- Compare safe and suspicious URLs.
- Run a phishing spotting exercise with mock examples.
- Build a one-page personal cyber hygiene protocol.

### Assessment Questions
1. Why is MFA important even if a password is strong?
2. List three phishing warning signs.
3. What is cyber hygiene?
4. Why is password reuse dangerous?
5. Write a 7-point personal digital safety checklist.

## CYB201 — Network Security Essentials

| Item | Details |
|---|---|
| **Course Code** | CYB201 |
| **Course Title** | Network Security Essentials |
| **Focus** | Firewalls, IDS/IPS, segmentation, access control, VPNs, and wireless security fundamentals |

### Course Purpose

This course introduces the core defensive technologies and design principles used to protect network environments. Learners begin to understand not just how networks function, but how they are monitored, controlled, segmented, and secured.

### Learning Outcomes

Learners should be able to:
- explain the purpose of network security
- define and compare firewalls, IDS, and IPS
- explain network segmentation and why it reduces risk
- describe access control at the network level
- explain the role of VPNs in secure remote access
- identify basic wireless security practices

### Core Concepts

#### 1. Firewalls
Firewalls inspect and filter traffic based on predefined rules. They help enforce security boundaries between trusted and untrusted networks.

#### 2. IDS and IPS
- **IDS** detects suspicious traffic and raises alerts
- **IPS** can detect and actively block malicious traffic

#### 3. Network Segmentation
Segmentation divides a network into smaller sections so that compromise in one area does not automatically expose everything else.

#### 4. Access Control
Network access control limits which users, devices, or systems can reach certain resources.

#### 5. VPN Fundamentals
Virtual Private Networks help protect remote communications by encrypting traffic between endpoints.

#### 6. Wireless Security
Learners should understand:
- secure Wi-Fi passwords
- trusted encryption standards
- avoiding open or poorly secured access points
- limiting unauthorized device access

### Practical Activities
- Draw a segmented office network with departments separated logically.
- Create a simple firewall rule table with allowed and blocked traffic examples.
- Compare IDS and IPS using short scenario examples.
- Map remote staff access through a VPN into an internal network.
- Identify weak vs strong wireless security configurations.

### Lab Tasks
- Create a simple secure network topology.
- Label trusted, semi-trusted, and untrusted zones.
- Document where a firewall, IDS/IPS, and VPN would be placed.

### Assessment Questions
1. What is the difference between a firewall and an IDS?
2. Why is segmentation important in a secure network?
3. What does a VPN do?
4. How does an IPS differ from an IDS?
5. Mention four wireless security best practices.

## CYB202 — System Hardening & Endpoint Security

| Item | Details |
|---|---|
| **Course Code** | CYB202 |
| **Course Title** | System Hardening & Endpoint Security |
| **Focus** | Updates, account policies, permissions, services management, antivirus, logging, and baseline controls |

### Course Purpose

This course teaches learners how to reduce the attack surface of operating systems and endpoints through secure configuration and disciplined maintenance.

### Learning Outcomes

Learners should be able to:
- explain what system hardening means
- identify insecure default configurations
- apply account and password policy basics
- manage permissions and services more securely
- explain the role of antivirus and endpoint protection
- understand logging and baseline control concepts

### Core Concepts

#### 1. System Hardening
System hardening means reducing exposure by disabling unnecessary features, strengthening configuration, and improving defensive controls.

#### 2. Updates and Patch Management
Patching helps close known weaknesses and keep systems aligned with supported security fixes.

#### 3. Account Policies
Strong account policy includes:
- password requirements
- account lockout settings
- least privilege
- review of inactive users

#### 4. Permissions and Services
Overly broad permissions and unnecessary running services increase exposure and should be reviewed regularly.

#### 5. Antivirus and Endpoint Protection
Endpoint tools help detect malicious files, suspicious processes, and abnormal behavior.

#### 6. Logging and Baselines
Security teams rely on logs and baseline configurations to notice changes, investigate issues, and verify secure system states.

### Practical Activities
- Review a Windows or Linux machine for unnecessary services.
- List hardening actions for a new workstation.
- Compare admin accounts and standard user accounts.
- Create a baseline checklist for a secure endpoint.
- Identify examples of useful system logs.

### Lab Tasks
- Produce a hardened endpoint checklist.
- Document password policy settings for a sample system.
- Record disabled services and explain why they were reviewed.
- Create a before-and-after hardening summary.

### Assessment Questions
1. What is system hardening?
2. Why are default settings often risky?
3. What is least privilege?
4. Why does logging matter in endpoint security?
5. Give five examples of basic hardening controls.

### Core Concepts

#### 2. Authorization

#### 3. Least Privilege

## CYB301 — Ethical Hacking & Penetration Testing

| Item | Details |
|---|---|
| **Course Code** | CYB301 |
| **Course Title** | Ethical Hacking & Penetration Testing |
| **Focus** | Reconnaissance, scanning, enumeration, exploitation awareness, post-exploitation awareness, and reporting |

### Course Purpose

This course introduces learners to the ethical hacking lifecycle in a controlled, legal, and academically guided context. Students learn how security professionals think, how assessments are staged, and how findings are translated into remediation guidance.

### Learning Outcomes

Learners should be able to:
- explain the purpose of ethical hacking
- distinguish legal authorized testing from malicious intrusion
- describe the major phases of a penetration test
- explain reconnaissance, scanning, and enumeration
- understand exploitation and post-exploitation at a high level
- document findings in a structured way

### Core Concepts

#### 1. Ethical Hacking vs Malicious Hacking
Ethical hacking is authorized security testing carried out to improve defense. Scope, permission, documentation, and professionalism are essential.

#### 2. Reconnaissance
Reconnaissance is the information-gathering phase. Testers collect data about hosts, services, domains, technologies, and exposed assets.

#### 3. Scanning
Scanning identifies live systems, open ports, services, and exposed points that may require deeper review.

#### 4. Enumeration
Enumeration goes further by identifying useful service details such as usernames, versions, shares, or application behavior.

#### 5. Exploitation Awareness
Students should understand that weaknesses may be abused if left unpatched or misconfigured. The educational focus here is on recognizing risk and validating impact safely within lab boundaries.

#### 6. Post-Exploitation Awareness
Students should understand why persistence, privilege escalation, credential exposure, and data access matter to defenders, without turning the class into unsafe instruction.

#### 7. Reporting
A penetration test is incomplete without a clear report. Reports should explain what was found, why it matters, evidence observed, and how to fix it.

### Practical Activities
- Build a diagram of the penetration testing lifecycle.
- Compare passive reconnaissance and active reconnaissance at a conceptual level.
- Create sample findings for an exposed web server or weak password policy.
- Practice writing a short severity statement and remediation note.

### Lab Tasks
- Walk through a controlled target profile in a lab.
- Record observed services and exposed areas.
- Map findings to likely business impact.
- Prepare a mini summary report.

### Assessment Questions
1. What makes ethical hacking ethical?
2. What is the difference between scanning and enumeration?
3. Why is scope important in penetration testing?
4. Why does reporting matter as much as testing?
5. List the main phases of a basic ethical hacking workflow.

## CYB302 — Tools for Vulnerability Scanning

| Item | Details |
|---|---|
| **Course Code** | CYB302 |
| **Course Title** | Tools for Vulnerability Scanning |
| **Focus** | Nmap, Nessus, OpenVAS, Metasploit awareness, Burp Suite, and Nikto in controlled lab environments |

### Course Purpose

This course introduces the major tools commonly referenced in vulnerability assessment and beginner lab-based security testing. The emphasis is on tool purpose, responsible use, interpreting results, and understanding how scanning supports remediation.

### Learning Outcomes

Learners should be able to:
- explain the purpose of common vulnerability assessment tools
- distinguish network discovery tools from vulnerability scanners
- interpret basic scan output responsibly
- identify web application testing tools conceptually
- explain how scan findings support remediation priorities
- compare automated tool output with manual validation needs

### Core Concepts

#### 1. Nmap
Nmap is used to discover hosts, ports, and service exposure. It helps testers understand the visible attack surface of a target.

#### 2. Nessus and OpenVAS
These tools automate vulnerability identification by checking systems against known weaknesses, unsafe configurations, and exposed services.

#### 3. Burp Suite
Burp Suite is widely used for web application inspection, request/response review, and structured web assessment workflows.

#### 4. Nikto
Nikto is commonly used to review web servers for common misconfigurations and known weak points.

#### 5. Metasploit Awareness
Students should know Metasploit as a framework often discussed in security labs and research. The educational focus here is awareness, controlled validation concepts, and understanding how findings may be tested in sandboxed environments.

#### 6. Interpreting Results
Automated scan results should not be treated as unquestionable truth. Findings often need verification, prioritization, and context.

### Practical Activities
- Compare network discovery output and vulnerability scan output.
- Review a sample scan report and classify findings by severity.
- Identify false positives and discuss why validation matters.
- Match each tool to the type of problem it is best suited for.

### Lab Tasks
- Use safe classroom examples of scan outputs.
- Build a findings table with columns for asset, issue, severity, evidence, and remediation.
- Compare web and network assessment tools in a worksheet.

### Assessment Questions
1. What is Nmap typically used for?
2. Why are Nessus and OpenVAS useful?
3. What type of work is Burp Suite associated with?
4. Why should scan results be validated?
5. What is the difference between discovery and vulnerability assessment?

## CYB303 — Security Testing Methodology

| Item | Details |
|---|---|
| **Course Code** | CYB303 |
| **Course Title** | Security Testing Methodology |
| **Focus** | Scoping, rules of engagement, documentation, severity communication, and remediation guidance |

### Course Purpose

This course teaches learners that good security testing is not random. It is planned, authorized, documented, and communicated professionally. Students learn how to define test scope, avoid unsafe behavior, and present findings in a way that organizations can act on.

### Learning Outcomes

Learners should be able to:
- define scope clearly for a security test
- explain rules of engagement
- document evidence consistently
- rate findings using clear severity language
- write remediation-oriented observations
- communicate professionally with technical and non-technical audiences

### Core Concepts

#### 1. Scope
Scope defines what systems, applications, or environments may be tested and what must remain excluded.

#### 2. Rules of Engagement
Rules of engagement set expectations for timing, permissions, escalation paths, allowed activities, and safety limits.

#### 3. Evidence Documentation
Good evidence includes screenshots, timestamps, affected assets, observed behavior, and concise descriptions.

#### 4. Severity and Risk
Students should learn how security issues are prioritized based on likelihood, impact, exploitability, and business relevance.

#### 5. Remediation Communication
A good report does not only describe problems. It recommends fixes, compensating controls, and next steps.

#### 6. Professionalism
Strong assessors are careful, accurate, respectful of scope, and clear in communication.

### Practical Activities
- Draft a sample scope statement for a web app lab.
- Write sample rules of engagement for a classroom exercise.
- Practice turning rough notes into proper evidence entries.
- Compare weak reporting language against strong remediation-focused wording.

### Lab Tasks
- Create a mini test plan.
- Build a severity matrix for classroom findings.
- Produce one sample executive summary and one technical finding entry.

### Assessment Questions
1. What is meant by test scope?
2. Why are rules of engagement important?
3. What should evidence include?
4. Why must findings be prioritized?
5. What makes a security report useful to an organization?

## CYB401 — Introduction to Cryptography

| Item | Details |
|---|---|
| **Course Code** | CYB401 |
| **Course Title** | Introduction to Cryptography |
| **Focus** | Encryption, hashing, salting, digital signatures, certificates, symmetric and asymmetric cryptography |

### Course Purpose

This course introduces learners to the fundamental concepts that underpin modern data protection. Students explore how secrecy, authenticity, integrity, and trust are achieved using cryptographic techniques.

### Learning Outcomes

Learners should be able to:
- define cryptography and explain why it matters
- distinguish encryption from hashing
- explain what salting does and why it is important
- compare symmetric and asymmetric cryptography
- explain digital signatures at a conceptual level
- identify where cryptography appears in everyday technology

### Core Concepts

#### 1. What Cryptography Does
Cryptography helps protect information by making it unreadable to unauthorized people, proving that data has not been altered, and supporting trust between communicating parties.

#### 2. Encryption
Encryption transforms readable data into protected ciphertext. Only someone with the appropriate key should be able to recover the original information.

#### 3. Hashing
Hashing produces a fixed-length representation of data that helps verify integrity. It is not designed to be reversed into the original content.

#### 4. Salting
Salting adds unique randomness before hashing sensitive data like passwords, helping reduce the risk of attackers using precomputed lookup tables.

#### 5. Symmetric Cryptography
In symmetric cryptography, the same secret key is used to encrypt and decrypt information.

#### 6. Asymmetric Cryptography
In asymmetric cryptography, a public key and a private key work together. This enables secure key exchange, signatures, and trust models used on the web.

#### 7. Digital Signatures
Digital signatures help prove authenticity and integrity. They allow a recipient to verify that a message came from the expected source and was not altered.

### Practical Activities
- Compare encryption, hashing, and encoding using real examples.
- Create a concept map showing when to use hashing vs encryption.
- Identify everyday systems that use cryptography, such as HTTPS, banking apps, and secure messaging.
- Write a short explanation of why password storage should use hashing rather than encryption.

### Lab Tasks
- Demonstrate simple file encryption and decryption with a safe classroom tool.
- Generate example hashes for test strings and compare results.
- Show how a small change in input changes the hash output.
- Build a table comparing symmetric and asymmetric cryptography.

### Assessment Questions
1. What is the difference between encryption and hashing?
2. Why is salting important for password protection?
3. Compare symmetric and asymmetric cryptography.
4. What problem do digital signatures help solve?
5. Give three real-world examples of cryptography in daily life.

## CYB402 — Public Key Infrastructure (PKI)

| Item | Details |
|---|---|
| **Course Code** | CYB402 |
| **Course Title** | Public Key Infrastructure (PKI) |
| **Focus** | SSL/TLS, HTTPS, certificate authorities, key exchange, VPN encryption, and trust chains |

### Course Purpose

This course explains how trust is established on modern networks and websites through certificates, public keys, and certificate authorities. Students learn why HTTPS works, how certificate chains are validated, and why PKI is central to secure communication.

### Learning Outcomes

Learners should be able to:
- explain what PKI means
- describe certificates and certificate authorities
- explain SSL/TLS and HTTPS conceptually
- understand basic key exchange ideas
- describe trust chains and certificate validation
- explain where PKI is used in VPNs, websites, and enterprise systems

### Core Concepts

#### 1. Public Key Infrastructure
PKI is the framework that supports the issuance, management, validation, and trust of digital certificates.

#### 2. Certificates
A certificate binds an identity to a public key. It helps systems know that the key they are using belongs to the party they expect.

#### 3. Certificate Authorities
Certificate authorities are trusted entities that issue and sign certificates.

#### 4. SSL/TLS and HTTPS
TLS protects data in transit between systems. HTTPS is the secure version of HTTP that typically relies on TLS.

#### 5. Trust Chains
Certificates are validated through a chain of trust from end-entity certificates to intermediate authorities and trusted roots.

#### 6. Key Exchange
Key exchange enables parties to establish secure session keys for encrypted communication.

#### 7. VPN Encryption
VPN technologies also rely on encryption and often certificates or shared secrets to protect communication tunnels.

### Practical Activities
- Inspect the certificate of a live HTTPS website in a browser.
- Identify the common name, issuer, validity dates, and trust chain.
- Compare HTTP and HTTPS at a conceptual level.
- Draw a diagram of how a browser validates a certificate.

### Lab Tasks
- Generate a sample key pair in a lab environment.
- Create a self-signed certificate for demonstration.
- Observe browser or tool behavior with trusted vs untrusted certificates.
- Document the certificate chain for a selected website.

### Assessment Questions
1. What is PKI?
2. What is the role of a certificate authority?
3. Why is HTTPS safer than HTTP?
4. What is a trust chain?
5. How do certificates support secure communication?

- explain why evidence handling must be careful and documented
- understand forensic imaging conceptually
- describe basic log- and artifact-based investigation methods
- recognize the difference between investigation and ordinary troubleshooting

### Core Concepts

#### 1. What Digital Forensics Means
Digital forensics is the process of identifying, preserving, examining, and interpreting digital evidence.

#### 2. Evidence Collection
Evidence may include logs, files, disk images, screenshots, timelines, memory artifacts, and account activity records.

#### 3. Chain of Custody
Chain of custody is the documented history of who handled evidence, when, how, and for what reason.

#### 4. Forensic Imaging
Forensic imaging creates a careful copy of storage media so examination can happen without altering the original.

#### 5. Integrity and Documentation
Investigations rely on accurate notes, timestamps, and evidence integrity so findings are trustworthy.

#### 6. Basic Analysis Methods
At beginner level, students should understand timelines, artifact review, log interpretation, and correlation of clues from different sources.

### Practical Activities
- Identify possible evidence sources in a mock incident.
- Fill out a sample chain-of-custody form.
- Compare ordinary file copying to the idea of forensic imaging.
- Build a simple investigation timeline from provided logs.

### Lab Tasks
- Review sample log files and highlight suspicious indicators.
- Document a mock evidence collection process.
- Create a timeline of user activity from supplied data.
- Write an evidence summary note with timestamps and findings.

### Assessment Questions
1. What is digital forensics?
2. Why is chain of custody important?
3. What is forensic imaging?
4. Why must investigators document their actions carefully?
5. Name four possible sources of digital evidence.

- interpret basic dashboards and security alerts

### Core Concepts

#### 1. What a SIEM Does
A SIEM collects logs from many sources, organizes them, and helps analysts search, correlate, and respond to suspicious activity.

#### 2. Data Sources
Common SIEM sources include firewalls, endpoints, servers, identity systems, applications, VPN gateways, and cloud services.

#### 3. Correlation
Correlation links multiple related events together so a system can identify suspicious patterns more effectively.

#### 4. Dashboards
Dashboards provide visual summaries of activity, trends, alert volume, and priority issues.

#### 5. Alert Prioritization

#### 6. Suspicious Behavior Analysis

## CYB601 — Cloud Security & Zero-Trust Architecture

| Item | Details |
|---|---|
| **Course Code** | CYB601 |
| **Course Title** | Cloud Security & Zero-Trust Architecture |
| **Focus** | Cloud responsibility models, IAM in cloud systems, cloud misconfigurations, zero-trust principles, and modern defensive architecture |

### Course Purpose

This course introduces learners to the security realities of cloud computing. Students explore how cloud environments are protected, what customers are responsible for, how identity and permissions shape cloud risk, and why zero-trust thinking is central to modern enterprise defense.

### Learning Outcomes

Learners should be able to:
- explain the shared responsibility model in cloud environments
- identify common cloud security risks and misconfigurations
- describe identity and access management in cloud systems
- explain zero-trust principles in simple terms
- understand why visibility, policy, and least privilege matter in cloud defense
- describe basic controls used in modern enterprise cloud security

### Core Concepts

#### 1. Shared Responsibility Model
Cloud providers secure parts of the environment, while customers remain responsible for properly configuring identities, data access, workloads, and many security settings.

#### 2. Cloud Misconfigurations
Many cloud incidents happen because services are left exposed, overly permissive, or poorly monitored.

#### 3. Cloud IAM
Identity and access management controls who can access resources, what they can do, and under what conditions.

#### 4. Zero-Trust Principles
Zero trust means users, devices, and systems should not be automatically trusted just because they are inside a network. Access should be continuously verified and tightly controlled.

#### 5. Least Privilege in Cloud Systems
Permissions should be narrow, specific, and reviewed often.

#### 6. Visibility and Monitoring
Logs, alerts, and security posture reviews are essential in cloud defense.

### Practical Activities
- Compare on-premises security thinking with cloud security thinking.
- Review sample cloud misconfiguration scenarios and recommend fixes.
- Create a simple shared responsibility matrix for a cloud-hosted application.
- Design a basic zero-trust access flow for staff, contractors, and administrators.

### Lab Tasks
- Build a cloud security checklist for a small business workload.
- Review example IAM roles and identify excessive privileges.
- Draft security controls for storage, compute, accounts, and logging.
- Create a one-page zero-trust architecture summary diagram.

### Assessment Questions
1. What is the shared responsibility model?
2. Why are cloud misconfigurations dangerous?
3. What is IAM in cloud security?
4. What does zero trust mean?
5. Why is least privilege especially important in cloud environments?
